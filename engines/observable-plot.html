Observable Plot Master Showcase

Comprehensive exploratory data visualization — every mark, transform, facet, and interaction

TipThe Grammar of Graphics in the Browser

Observable Plot (Plot) is a declarative JavaScript library for exploratory data visualization. It combines composable marks, data transforms (binning, grouping, stacking, dodging, density), faceted small multiples, and interactive tooltips (Plot.tip()) into a concise grammar of graphics interface that executes natively in Quarto without server reliance.

1. Scatterplots & Beeswarm Dodge (Plot.dot, Plot.dodgeX)

Scatterplots assign numerical attributes to \(X\) and \(Y\) dimensions. When overplotting occurs, Plot.dodgeX() or Plot.dodgeY() arranges points into beeswarm distributions without overlap.

Code 
// Simulated patient sample dataset
patients = d3.range(120).map(i => {
  const genotype = ["HbSS", "HbS/β0", "HbS/β+", "HbSC"][i % 4];
  const sex = i % 2 === 0 ? "Female" : "Male";
  const hb = genotype === "HbSS" ? 6.5 + Math.random() * 2.5 :
             genotype === "HbS/β0" ? 7.2 + Math.random() * 2.2 :
             genotype === "HbS/β+" ? 9.0 + Math.random() * 2.5 :
             10.5 + Math.random() * 2.5;
  const hbf = genotype === "HbSS" ? 8 + Math.random() * 14 :
              genotype === "HbS/β0" ? 5 + Math.random() * 10 :
              1 + Math.random() * 5;
  const retic = 120 + (15 - hb) * 25 + (Math.random() - 0.5) * 40;
  return { id: `P-${100+i}`, genotype, sex, hb, hbf, retic };
})
Code 
Plot.plot({
  title: "Hemoglobin (Hb) vs Fetal Hemoglobin (HbF) by Genotype",
  width: 760,
  height: 380,
  grid: true,
  x: {label: "Hemoglobin Hb (g/dL) →"},
  y: {label: "↑ Fetal Hemoglobin HbF (%)"},
  color: {legend: true},
  marks: [
    Plot.ruleY([0]),
    Plot.dot(patients, {
      x: "hb",
      y: "hbf",
      fill: "genotype",
      r: 4.5,
      fillOpacity: 0.8,
      tip: true
    })
  ]
})
Code 
Plot.plot({
  title: "Beeswarm Distribution of Reticulocyte Count across Genotypes",
  width: 760,
  height: 280,
  x: {label: "Genotype"},
  y: {label: "Reticulocyte Count (x10⁹/L)", grid: true},
  color: {legend: true},
  marks: [
    Plot.dot(patients, Plot.dodgeX("dot", {
      x: "genotype",
      y: "retic",
      fill: "genotype",
      r: 4,
      tip: true
    }))
  ]
})

2. Histograms & Automated Binning (Plot.rectY, Plot.binX)

The Plot.binX() transform automatically bins continuous numerical data and counts occurrences:

Code 
Plot.plot({
  title: "Hemoglobin Frequency Histogram",
  width: 760,
  height: 320,
  grid: true,
  x: {label: "Hemoglobin Hb (g/dL)"},
  y: {label: "↑ Patient Count"},
  color: {legend: true},
  marks: [
    Plot.rectY(patients, Plot.binX({y: "count"}, {x: "hb", fill: "sex", thresholds: 15})),
    Plot.ruleY([0])
  ]
})

9. Layered marks & curve interpolation (Plot.areaY, lineY, curve)

Plot has no chart types — you layer marks. A time-series below layers a translucent area, a line, and a ruleY baseline; a second panel shows how the curve option changes interpolation (linear vs. catmull-rom vs. step).

Code 
// Synthetic ferritin trajectory over 24 weeks of iron chelation
series = d3.range(0, 25).map(w => ({
  week: w,
  ferritin: 2400 * Math.exp(-0.06 * w) + 120 * Math.sin(w / 2) + (d3.randomNormal(0, 60)())
}))
Code 
Plot.plot({
  title: "Serum ferritin during iron chelation",
  width: 760, height: 320, grid: true,
  x: {label: "Weeks on chelation →"},
  y: {label: "↑ Ferritin (µg/L)"},
  marks: [
    Plot.ruleY([500], {stroke: "#2e7d32", strokeDasharray: "4"}),
    Plot.areaY(series, {x: "week", y: "ferritin", fill: "#B71C1C", fillOpacity: 0.12}),
    Plot.lineY(series, {x: "week", y: "ferritin", stroke: "#B71C1C", strokeWidth: 2.2, tip: true})
  ]
})
Code 
viewof curveType = Inputs.radio(["linear", "catmull-rom", "step", "basis"], {value: "catmull-rom", label: "Curve interpolation:"})
Code 
Plot.plot({
  width: 760, height: 220, grid: true,
  x: {label: "Weeks →"}, y: {label: "Ferritin (µg/L)"},
  marks: [
    Plot.lineY(series, {x: "week", y: "ferritin", curve: curveType, stroke: "#7f0000", strokeWidth: 2.4}),
    Plot.dot(series, {x: "week", y: "ferritin", r: 2, fill: "#7f0000"})
  ]
})

10. Stacked areas & streamgraphs (stack, offset)

The areaY/barY marks apply an implicit stack transform. Setting offset: "wiggle" produces a streamgraph — ideal for cohort composition over time.

Code 
composition = d3.range(0, 25).flatMap(m =>
  ["HbSS", "HbS/β0", "HbS/β+", "HbSC"].map((g, i) => ({
    month: m, genotype: g,
    n: 20 + 8 * Math.sin(m / 4 + i) + i * 6 + d3.randomNormal(0, 3)()
  })))
Code 
Plot.plot({
  title: "Clinic cohort composition over time (streamgraph)",
  width: 760, height: 300,
  x: {label: "Month →"}, y: {label: null},
  color: {legend: true, scheme: "Reds"},
  marks: [
    Plot.areaY(composition, {x: "month", y: "n", fill: "genotype", offset: "wiggle", curve: "basis", tip: true})
  ]
})

11. Rolling averages & normalisation (Plot.windowY, Plot.normalizeY)

The window transform smooths a noisy signal (moving average); the normalize transform re-indexes series to a common baseline for fair comparison.

Code 
Plot.plot({
  title: "Reticulocyte signal: raw vs 5-week moving average",
  width: 760, height: 300, grid: true,
  x: {label: "Weeks →"}, y: {label: "Reticulocytes"},
  marks: [
    Plot.lineY(series, {x: "week", y: "ferritin", stroke: "#ccc", strokeWidth: 1}),
    Plot.lineY(series, Plot.windowY({k: 5, reduce: "mean"}, {x: "week", y: "ferritin", stroke: "#B71C1C", strokeWidth: 2.5}))
  ]
})

12. Parts of a whole (Plot.waffleY)

The waffle mark renders counts as unit squares — a friendlier alternative to pie charts for treatment allocation.

Code 
allocation = [

  {arm: "Factor VIII prophylaxis", n: 41},
  {arm: "On-demand", n: 18}

Code 
Plot.plot({

  width: 760, height: 220,
  color: {legend: true, scheme: "Reds"},
  marks: [Plot.waffleY(allocation, {x: "arm", y: "n", fill: "arm"})]

13. Hexbin density (Plot.hexbin)

The hexbin transform aggregates 2-D points into hexagonal bins — a scalable alternative to overplotted scatterplots.

Code 
Plot.plot({
  title: "Hexbin density (Hb vs reticulocytes)",
  width: 760, height: 360, grid: true,
  x: {label: "Hemoglobin (g/dL) →"}, y: {label: "↑ Reticulocytes"},
  color: {scheme: "Reds", legend: true, label: "count"},
  marks: [
    Plot.dot(patients, Plot.hexbin({r: "count", fill: "count"}, {x: "hb", y: "retic"})),
    Plot.frame()
  ]
})

14. Change & fields (Plot.arrow, Plot.vector)

The arrow mark connects a start and end point (e.g. before/after treatment); the vector mark draws little arrows for magnitude + direction fields.

Code 
prePost = ["HbSS", "HbS/β0", "HbS/β+", "HbSC"].map((g, i) => ({
  genotype: g, hbfBefore: [8, 5, 3, 2][i], hbfAfter: [22, 16, 9, 6][i]

Code 
Plot.plot({
  title: "HbF response to hydroxycarbamide (before → after)",

16. Comparison (Plot.differenceY)

The difference mark fills the gap between two series, colouring gains and shortfalls differently — a natural fit for observed vs target trough levels.

Code 
troughs = d3.range(0, 25).map(w => ({
  week: w, observed: 45 + 15 * Math.sin(w / 3) + d3.randomNormal(0, 4)(), target: 50
}))
Code 
Plot.plot({
  title: "Observed trough vs 50% target",
  width: 760, height: 300, grid: true,
  x: {label: "Weeks →"}, y: {label: "Factor level (%)"},
  marks: [
    Plot.differenceY(troughs, {x: "week", y1: "target", y2: "observed", positiveFill: "#2e7d32", negativeFill: "#B71C1C"}),
    Plot.lineY(troughs, {x: "week", y: "observed", stroke: "black", strokeWidth: 1.2})
  ]
})

18. Maps & projections (Plot.geo, Plot.sphere, Plot.graticule)

Plot speaks GeoJSON and D3 projections natively. This self-contained globe uses an orthographic projection with a sphere, graticule, and plotted centres — no external shapefiles required.

Code 
centres = [
  {name: "Bangor", lon: -4.13, lat: 53.23},
  {name: "London", lon: -0.13, lat: 51.51},
  {name: "Paris (ISTH)", lon: 2.35, lat: 48.86},
  {name: "New York", lon: -74.0, lat: 40.71}
]
Code 
Plot.plot({
  width: 480, height: 480,
  projection: {type: "orthographic", rotate: [10, -40]},
  marks: [
    Plot.sphere({fill: "#f7f3f3", stroke: "#ddd"}),
    Plot.graticule({stroke: "#e2c9c9"}),
    Plot.dot(centres, {x: "lon", y: "lat", fill: "#B71C1C", r: 5}),
    Plot.text(centres, {x: "lon", y: "lat", text: "name", dy: -10, fontWeight: 600, fill: "#7f0000"})
  ]
})

19. Continuous fields (Plot.raster, Plot.contour)

The raster and contour marks visualise a scalar field over a 2-D domain — here an illustrative “risk surface” as a function of two lab values.

Code 
Plot.plot({
  title: "Illustrative risk surface f(Hb, platelets)",
  width: 760, height: 340,
  x: {label: "Hemoglobin (g/dL) →"}, y: {label: "↑ Platelets (×10⁹/L)"},
  color: {scheme: "Reds", legend: true, label: "risk"},
  marks: [
    Plot.raster({
      x1: 4, x2: 16, y1: 0, y2: 400,
      fill: (x, y) => Math.max(0, (14 - x) / 10) + Math.max(0, (50 - y) / 50)
    }),
    Plot.contour({
      x1: 4, x2: 16, y1: 0, y2: 400,
      fill: (x, y) => Math.max(0, (14 - x) / 10) + Math.max(0, (50 - y) / 50),
      stroke: "white", strokeOpacity: 0.4
    })
  ]
})

20. The auto mark (Plot.auto)

Plot.auto inspects your data and picks a sensible mark and transform — the fastest way to “just show me the data”.

Code 
Plot.plot({
  height: 300,
  marks: [Plot.auto(patients, {x: "hb", y: "retic", color: "genotype"})]
})

21. Interaction: crosshair, and a reactive Plot

The crosshair mark reads out the nearest value on the axes without obscuring the data. And because a Plot is just a value, a viewof input can rebuild it live — the pattern behind interactive dashboards.

Code 
viewof minHbF = Inputs.range([0, 20], {value: 0, step: 1, label: "Show patients with HbF ≥ (%)"})
Code 
Plot.plot({
  width: 760, height: 360, grid: true,
  x: {label: "Hemoglobin (g/dL) →"}, y: {label: "↑ Reticulocytes"},
  color: {legend: true},
  marks: [
    Plot.dot(patients.filter(d => d.hbf >= minHbF), {x: "hb", y: "retic", stroke: "genotype"}),
    Plot.crosshair(patients.filter(d => d.hbf >= minHbF), {x: "hb", y: "retic"})
  ]
})

TipOne grammar, every figure
